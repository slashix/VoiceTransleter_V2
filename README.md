# VoiceTransleter V2 (CUDA fork)

Дубляж видео с переводом речи на русский язык с синтезом голоса (любой язык → RU).

**Форк версии:** 2.1.7, с существенными доработками под GPU-инференс и качество перевода/синхронизации.

**Оригинальный проект:** [AndranikFutureLabs/VoiceTransleter_V2](https://github.com/AndranikFutureLabs/VoiceTransleter_V2)
**Оригинальный автор:** Андраник Алавердян ([@AndranikFutureLabs](https://t.me/AndranikFutureLabs))

**Разработка форка:** код изменений в этом форке написан с помощью Claude Sonnet 5

## О программе

VoiceTransleter V2 — десктопное приложение для автоматического дубляжа видео. Распознаёт речь на любом языке (Whisper), переводит на русский, синтезирует голос (XTTS v2 с клонированием голоса) и встраивает новую аудиодорожку в видео.

Этот форк переориентирует пайплайн на **CUDA-инференс** (вместо CPU), **платный API-переводчик с сохранением контекста** (вместо построчного Google Translate) и **посегментную синхронизацию тайминга** (вместо грубого глобального time-stretch всей дорожки), плюс устраняет несколько практических проблем, найденных при реальном использовании на технических видео.

## Что изменено в этом форке

Все изменения найдены и проверены на реальных 1-10-минутных технических видео (сетевые технологии — BGP, EVPN, VXLAN), с оригиналом на английском и переводом на русский.

### 1. CUDA вместо CPU (Whisper + XTTS)
Оригинал жёстко фиксировал `device="cpu", compute_type="int8"` для faster-whisper и `gpu=False` для XTTS. Форк:
- Whisper: `large-v3` на CUDA (`float16`) вместо `medium` на CPU — точнее распознаёт технические термины, на порядок быстрее
- XTTS: перенесена на GPU, с автоматическим fallback на CPU, если CUDA недоступна или инициализация не удалась
- И то, и другое управляется через переменные окружения (`WHISPER_MODEL`, `WHISPER_DEVICE`, `TTS_DEVICE`) без правки кода

### 2. DeepSeek вместо Google Translate
Оригинал использовал `google-translate-api-x` — построчный перевод без ключа, без контекста между соседними репликами, что давало нестыковки терминологии на длинных видео. Форк:
- Перевод через DeepSeek API (модель `deepseek-v4-flash`) с системным промптом под технический перевод для озвучки
- **Batch-перевод с сохранением контекста**: все сегменты видео переводятся ОДНИМ запросом с маркерами `[[[PART_N]]]`, а не по одному — модель видит полный текст сразу, что даёт согласованную терминологию по всему видео. При сбое разметки — автоматический откат на построчный перевод для проблемного батча
- Retry с exponential backoff на 503/429 (временная перегрузка API)

### 3. Per-segment синхронизация вместо глобального time-stretch
Оригинал растягивал/сжимал ВСЮ аудиодорожку одним общим коэффициентом под длительность видео — при разном темпе речи в разных местах это давало заметный рассинхрон. Форк подгоняет **каждый сегмент индивидуально** под его тайм-слот:
- Короче слота (≤15% разницы) — растяжение через `atempo`
- Короче слота (>15%) — растяжение до предела, остаток компенсируется тишиной
- Длиннее слота — сжатие `atempo`, максимум 1.5x; если не хватает — сдвиг последующих сегментов (без обрезки речи)

### 4. Устранение Whisper-галлюцинаций
На монотонных участках аудио (задержки, паузы) faster-whisper иногда "залипает" и генерирует один и тот же текст десятки раз подряд на почти нулевых тайм-слотах — это давало до 100+ секунд паразитного дрейфа синхронизации на 10-минутном видео. Форк детектирует 3+ идущих подряд идентичных сегмента и схлопывает их в один с правильным таймингом.

### 5. Склейка рублёных фраз
Whisper иногда режет одну фразу на несколько сегментов по внутренним порогам пауз (диктор сделал вдох посреди предложения) — это давало рубленую, неестественную по интонации озвучку на границах. Форк склеивает такие соседние сегменты обратно в одну фразу перед синтезом (по паузе и отсутствию финальной пунктуации).

### 6. Дробление длинных сегментов под лимит XTTS
XTTS v2 нестабильно ведёт себя на тексте длиннее ~182 символов для русского (архитектурное ограничение модели, зависит от языка — не общие "250"). Форк дробит длинные сегменты строго по границам предложений/запятых перед синтезом, не обрезая текст молча.

### 7. Словарь произношения технических акронимов
XTTS нестабильно произносит акронимы как единое слово (например, "BGP" может звучать искажённо). Форк заменяет известные акронимы на слитную транслитерацию, как их реально произносят вслух (`BGP` → "биджипи", `VXLAN` → "веикслан" и т.д.) — дополняемый словарь в отдельном файле `acronyms.ts`.

### 8. Технические фиксы окружения
- Миграция с мёртвого `TTS` (Coqui AI закрылась в 2024) на поддерживаемый форк `coqui-tts`
- Совместимость с `transformers` (зафиксирован на `4.57.6` — более новые ломают `coqui-tts`)
- Поддержка `torchcodec` (обязателен для PyTorch 2.9+)
- Убран Windows-специфичный хардкод путей

## Возможности (обновлено)

- **Распознавание речи** — faster-whisper (модель large-v3, CUDA, float16, с fallback на CPU)
- **Автоопределение языка** оригинала
- **Перевод на русский** — DeepSeek API, batch-режим с сохранением контекста
- **Синтез речи** — XTTS v2 (Coqui TTS, форк) с клонированием голоса по WAV-образцу, на CUDA
- **Per-segment синхронизация** — индивидуальная подгонка каждого сегмента под его тайм-слот
- **Устранение ASR-галлюцинаций** и **склейка рублёных фраз** перед переводом/синтезом
- **Словарь произношения акронимов** — дополняемый, под технический контент
- **Сохранение текстов** — 8 файлов: SRT с таймингами и plain, исходный текст, перевод, транслитерация
- **Голосовые профили** — сохранение нескольких образцов голоса
- **Трей** — свёртка в системный лоток

## Установка

### Требования

- Node.js 20+
- Python 3.12 (проверено; более старые тоже должны работать)
- NVIDIA GPU с CUDA (проверено на RTX 5080, CUDA 13.0) — без GPU работает на CPU, но заметно медленнее
- Git
- FFmpeg

### Сборка из исходников (Linux, проверено на Ubuntu 24.04)

```bash
git clone <URL-твоего-форка>
cd VoiceTransleter_V2

# Node.js зависимости
npm install

# Python окружение — ОТДЕЛЬНЫЙ venv, не системный Python
python3 -m venv .venv
source .venv/bin/activate

# PyTorch под твою версию CUDA (проверь: nvidia-smi | grep "CUDA Version")
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu130

# TTS-движок (форк, НЕ оригинальный мёртвый пакет TTS)
pip install coqui-tts[codec]
pip install "transformers==4.57.6"  # более новые версии несовместимы, см. idiap/coqui-ai-TTS#558

# Whisper
pip install faster-whisper ctranslate2

# Явно укажи интерпретатор venv (иначе платформенный автопоиск может
# найти неподходящую версию Python в системе)
export VOICE_TRANSLATOR_PYTHON=$(pwd)/.venv/bin/python3

# DeepSeek API ключ (platform.deepseek.com)
export DEEPSEEK_API_KEY=твой_ключ

# Запуск GUI
npm run dev
```

## Использование

> ⚠️ **Обязательно загрузи образец голоса перед первым запуском дубляжа.** Без него XTTS v2 использует нестабильную заглушку — результат будет звучать РАЗНЫМ голосом на каждом сегменте, местами шумом вместо речи. Это не баг, а особенность модели: XTTS требует явный референс-голос для клонирования, без него не может дать стабильный результат. Загрузи WAV с чистой речью (6-10 секунд, без фоновой музыки/шума) через кнопку **«Добавить голос»** до того, как начнёшь обработку видео.

### Первый запуск

1. Запусти VoiceTransleter V2 (`npm run dev` или собранный бинарник)
2. Дождись автоматической проверки/загрузки зависимостей — Whisper large-v3 (~3 ГБ) и XTTS v2 (~2.5 ГБ) скачиваются один раз
3. Статус моделей в интерфейсе станет зелёным
4. **Загрузи образец голоса** через «Добавить голос» (см. предупреждение выше)

### Дубляж видео

1. Загрузи видеофайл
2. Выбери язык оригинала (или автоопределение)
3. Выбери голос — тот, что загрузил на шаге выше (не оставляй «По умолчанию», если не загрузил свой образец)
4. Нажми **«Начать дубляж»**
5. Этапы в логе:
   - **[1/5]** извлечение аудио
   - **[2/5]** распознавание речи (с дедупликацией галлюцинаций и склейкой рублёных сегментов)
   - **[3/5]** перевод (batch, с контекстом)
   - **[4/5]** синтез речи (с дроблением длинных сегментов и произношением акронимов)
   - **[5/5]** сборка видео (per-segment синхронизация)

### Результаты

Те же 9 файлов, что и в оригинале — см. таблицу в разделе "Результаты" ниже.

| Файл | Описание |
|------|----------|
| `*_dubbed.mp4` | Готовое видео с дубляжом |
| `*_source.txt` | Исходный текст (SRT) |
| `*_source_plain.txt` | Исходный текст без таймингов |
| `*_translation.txt` | Перевод на русский (SRT) |
| `*_translation_plain.txt` | Перевод без таймингов |
| `*_source_translit.txt` | Транслитерация оригинала (SRT) |
| `*_source_translit_plain.txt` | Транслитерация оригинала без таймингов |
| `*_translation_translit.txt` | Транслитерация перевода (SRT) |
| `*_translation_translit_plain.txt` | Транслитерация перевода без таймингов |

## Системные требования

- **ОС:** проверено на Ubuntu 24.04; должно работать на других Linux/Windows/macOS
- **GPU:** NVIDIA с поддержкой CUDA настоятельно рекомендуется (проверено на RTX 5080/CUDA 13.0); без GPU работает на CPU, но медленно
- **ОЗУ:** 16+ ГБ рекомендуется
- **VRAM:** от 6 ГБ (Whisper + XTTS вместе занимают ~7 ГБ на CUDA)

- **Python:** 3.12 проверено (отдельный venv, НЕ системный)
- **DeepSeek API ключ:** platform.deepseek.com, недорогой (доли цента за минуту видео)

## Устранение неполадок

### `ImportError: cannot import name 'isin_mps_friendly'`
Несовместимость `transformers` с `coqui-tts`. Зафиксируй версию:
```bash
pip install "transformers==4.57.6"
```

### `ImportError ... torchcodec`
PyTorch 2.9+ требует отдельный пакет для аудио-IO:
```bash
pip install "coqui-tts[codec]"
```

### `Python 3.12.3 не поддерживается`
Это сообщение от устаревшей проверки версии в исходном коде — если видишь его на актуальной версии форка, значит патч `python_deps.ts` не применился (форк поддерживает 3.9–3.12 для `coqui-tts`).

### Голос звучит по-разному в каждом сегменте / как шум
Не загружен образец голоса — XTTS использует нестабильную заглушку. Это ожидаемое поведение модели, а не баг: **загрузи WAV с чистой речью (6-10 секунд, без фона) через «Добавить голос»** до начала дубляжа — см. предупреждение в начале раздела "Использование".

### `CUDA out of memory`
Проверь, не висят ли старые процессы `whisper_server.py`/`tts_server.py` от прошлых запусков:
```bash
nvidia-smi   # смотри Processes внизу
kill <PID>   # для зависших процессов
```

### Как проверить, что CUDA реально используется
В логе при загрузке моделей должно быть:
```
Loading faster-whisper large-v3 (cuda, float16)...
Loading XTTS v2 into memory (GPU, ...)
```
Если видишь `(cpu, ...)` вместо `cuda`/`GPU` — CUDA недоступна или упала инициализация (см. лог выше по тексту на предмет `CUDA load failed`).
